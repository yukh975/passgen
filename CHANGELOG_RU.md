# История изменений

## v1.0.1 — не выпущено

### Функционал
- **Вкладка «SSH-ключ»**: генерация пар ключей ED25519 и RSA (2048 / 3072 / 4096 бит) прямо в браузере через Web Crypto API; ключи экспортируются в стандартном формате OpenSSH (`-----BEGIN OPENSSH PRIVATE KEY-----`) с публичным ключом в формате `.pub`; кнопки копирования для приватного и публичного ключа
- **Скачать архив**: кнопка справа от панели действий (появляется после генерации) скачивает оба ключа одним ZIP-файлом (`<имя>.key` + `<имя>.pub`) с одинаковым случайным именем; ZIP создаётся на чистом JS без внешних библиотек
- **Проверить ключи**: модальное окно для проверки принадлежности приватного и публичного ключей одной паре — вставить или загрузить каждый ключ, поддерживается загрузка файлов (`.key`, `.pub`, `.pem`); полная поддержка ключей с паролем: приватная секция расшифровывается (bcrypt + AES-256-CTR) с проверкой контрольных значений (checkint) перед сравнением публичного ключа
- **Пароль для SSH-ключа**: необязательное поле пароля на вкладке SSH-ключ; при заполнении приватный ключ шифруется AES-256-CTR + bcrypt KDF (16 раундов) — формат идентичен выводу `ssh-keygen`, совместим со стандартными инструментами `ssh`/`ssh-keygen`; bcrypt реализован на чистом JS без внешних библиотек
- **Вкладка «GPG-ключ»**: генерация пар ключей OpenPGP v4 (Ed25519 или RSA 3072/4096 бит) прямо в браузере через Web Crypto API; ключи экспортируются в формате ASCII armor и импортируются командой `gpg --import`; необязательное поле «Идентификатор пользователя» в формате `Имя <email>`; кнопка «Скачать архив» сохраняет файлы `.asc` и `.pub.asc` одним ZIP; реализовано на чистом JS без внешних библиотек

### Исправления
- Верификация SSH-ключей: парсер формата OpenSSH переписан с использованием `DataView.getUint32` и построчным разбором PEM — устранена ошибка «Truncated key data»
- Высота textarea с ключами теперь вычисляется через количество строк (`rows`), а не `scrollHeight` — корректно работает при первой загрузке на скрытой вкладке
- Область результата и кнопка «Скачать архив» больше не генерируются автоматически при загрузке; вкладка стартует пустой и корректно очищается при сбросе настроек

## v1.0.0 — 2026-03-22

### Функционал
- **Вкладка «Пароль»**: слайдер длины (4–64, по умолчанию 20), переключатели наборов символов (заглавные, строчные, цифры, специальные), исключение похожих символов (I, l, 1, O, 0), запрет повторяющихся символов, поля доп. символов и исключений, 8 пресетов для платформ (PostgreSQL, MySQL, SQL Server, Linux User, Windows Admin, Redis, RabbitMQ, High Entropy), индикатор надёжности (шкала 1–5 с цветовой кодировкой)
- **Вкладка «Фраза»**: слайдер количества слов (2–10, по умолчанию 8), опции «С заглавной» / «Случайный регистр», добавление цифры в начало и/или конец, числа внутри фразы (случайно добавляет цифру к части слов), выбор разделителя (пробел, дефис, точка, подчёркивание, случайный символ, без разделителя; по умолчанию: дефис), встроенный словарь ~500 слов
- **Вкладка «PIN»**: слайдер длины (4–12 цифр, по умолчанию 6)
- Генерация нескольких результатов одновременно (слайдер количества 1–50) на всех трёх вкладках
- Кнопка копирования у каждого результата; кнопка «Скопировать все» открывает модальное окно со всеми результатами в textarea
- Кнопка «Пересоздать» — общая для всех результатов на каждой вкладке
- Результаты видны сразу (без переключателя показ/скрытие)
- Криптографически стойкая случайность через `window.crypto.getRandomValues()` с rejection sampling для устранения смещения по модулю
- Индикатор надёжности на основе энтропии Шеннона + разнообразие типов символов: «Сильный» требует 2+ типа, «Очень сильный» требует 3+ типа и ≥80 бит; пароли из одного типа символов ограничены уровнем «Средний» независимо от длины
- Кнопка сброса настроек (↺) очищает весь localStorage и перезагружает страницу
- Исправлена ошибка: при восстановлении настроек из localStorage значения, превышающие допустимый максимум, отображались в полях ввода без зажима
- Исправлено: числовое поле ввода больше не перезаписывается при каждом нажатии клавиши — зажатие происходит только при потере фокуса; набор многозначных чисел теперь работает корректно
- Сброс настроек теперь записывает чистый версионированный объект в localStorage, гарантируя возврат всех полей к значениям по умолчанию при перезагрузке

### Интерфейс
- Кнопка переключения тёмной/светлой темы (иконка солнца/луны) в заголовке; по умолчанию тёмная тема
- Переключатель языка EN / RU в заголовке и внутри окна справки
- Кнопка справки (?) с руководством пользователя
- SVG-логотип с изображением замка слева от заголовка страницы
- `favicon.ico` сгенерирован из логотипа (16/32/48 px, чистый Node.js, без внешних инструментов)
- Версия приложения в футере

### Сохранение настроек
- Все настройки генератора сохраняются в `localStorage` и восстанавливаются при перезагрузке: тема, язык, активная вкладка, все опции каждой вкладки (длина, количество, пресеты, чекбоксы, разделитель)
- Версионирование настроек с миграцией: устаревшие значения из ранних сборок автоматически исправляются при первой загрузке

# Passgen — Генератор надёжных паролей

Лёгкий клиентский генератор паролей с тремя режимами: **Пароль**, **Фраза-пароль** и **PIN**. Вся генерация происходит в браузере с использованием Web Crypto API — никакие данные не передаются на сервер.

**Демо:** *(скоро)*

---

## Возможности

### Пароль
- Настраиваемая длина (4–64 символа)
- Наборы символов: заглавные, строчные, цифры, специальные символы
- Исключение визуально похожих символов (I, l, 1, O, 0)
- Поля для дополнительных и исключаемых символов
- **8 пресетов для платформ**: PostgreSQL, MySQL, SQL Server, Linux User, Windows Admin, Redis, RabbitMQ, High Entropy
- Индикатор надёжности (Очень слабый → Очень сильный)

### Фраза-пароль
- Случайные наборы слов из словаря ~500 слов
- Настраиваемое количество слов (2–10)
- Варианты регистра: с заглавной буквы или случайный для каждой буквы
- Опциональная цифра в начале и/или конце
- Разделитель: пробел, дефис, точка, подчёркивание, случайный символ или без разделителя

### PIN
- Числовой код, настраиваемая длина (4–12 цифр)

### Общее
- Показ / скрытие результата
- Копирование одним кликом
- Кнопка пересоздать
- Тёмная / светлая тема (сохраняется в `localStorage`)
- Переключатель языка EN / RU
- Криптографически стойкая случайность через `window.crypto.getRandomValues()`
- Полностью клиентский — без сервера, без трекинга

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/yukh975/passgen.git
   cd passgen
   ```
2. Откройте `index.html` в браузере — сборка не требуется.

Или запустите любой статический сервер:
```bash
npx serve .
```

---

## Безопасность

- Вся случайность берётся из `window.crypto.getRandomValues()` с rejection sampling для устранения смещения по модулю.
- Данные не покидают браузер — генерация полностью локальная.
- Нет зависимостей от внешних CDN или удалённых ресурсов.
